## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can run `npm run dev` and demo what exists.
2. **Infrastructure before features.** Phase 0 sets up the stack, the zero-network guarantee, and the persistence layer so feature work is fast and clean.
3. **Magic moment first.** Batch signing — the reason this product exists — lands by the end of Phase 2, not in polish.
4. **Test as you go.** Every task's Notes end with a verification step. The zero-network Playwright check runs from Phase 0 onward.
5. **Progressive enhancement.** Simplest working version first (single doc, main-thread flatten), then layer on (batch, worker, zip).
6. **Review before proceeding.** Push each completed phase as a PR (`phase-{N}/{slug}`) and let a review agent (e.g. CodeRabbit) review before starting the next phase. If not using GitHub, at minimum run the full test suite and a manual smoke of the phase goal before moving on.
7. **Zero requests is a test, not a promise.** (Product-specific.) Any task that adds an asset must bundle it locally. The network assertion failing is a build-stopping event.
8. **The founder is the acceptance test.** (Product-specific.) Phase goals are verified against the founder's real documents, not synthetic samples.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

Works with any coding agent (Claude Code, Codex, Pi, Opencode — the founder uses several; prompts below are agent-agnostic).

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase.
2. **Read selectively:** Each phase lists its Reference sections — the specific parts of the PRD and vision doc needed for that phase. The agent should read only those sections, not the entire documents.
3. **Let it work:** The agent reads the roadmap, finds the first unchecked task, implements it, and marks it complete.
4. **One session = one phase (ideally):** Try to complete a full phase in one session for best continuity. If you need to stop, the agent can resume from the last unchecked task.
5. **Push a PR for review:** When a phase is complete, push the work as a PR and let a review agent (e.g. [CodeRabbit](https://coderabbit.ai)) review it before starting the next phase. If you skip GitHub/PRs, you give up automated review that catches bugs the coding agent won't flag — at minimum run the full test suite and manually verify the phase goal.
6. **Need more context?** If a task references a section not in the phase's Reference sections, the agent should read just that section on demand.

### Session Tips

- **Don't read everything:** The PRD and vision doc are large. Each phase's Reference sections tell the agent exactly what to read. Loading the full documents wastes context.
- **Don't skip tasks:** Tasks are ordered intentionally. Skipping creates dependency issues.
- **Verify after each phase:** Run the app after completing a phase to confirm everything works before moving on. The zero-network e2e is the canary — it must be green at every phase boundary.
- **Review before moving on:** Don't start the next phase until the phase PR is reviewed and merged (or the manual equivalent is done).
- **Update the status line:** After completing tasks, update the header: `**Status:** X/41 tasks complete` and `**Current Phase:** Phase N — {Title}`.
